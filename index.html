This page is an experiment with embedding interactive simulations into web pages. Make this change at 1:21:00 and try to see how fast it can be seen in github.io.

The objective is to combine reading and doing in one place to help students learn about complex physical systems.

This text is in a division with the css class "class_body_text". The css file has instructions on how to lay out this division.

Below is a prototype of a simulation that takes up most of the page width. It is in a class "class_simulation_frame" and it's own id. In the css file, the class specification can specify general characteristics of all simulations, then the id specification can set properties specific to this simulation

Now I want to look at a bigger simulation frame. Looks like the biggest you can have to fit in an iPhone 5s browser is approx. max width 958 and margin 2. Want position relative for all simulation frames. And want to make controls larger than the default size in LiveCode lcCardLayoutToWeb.



The outer group of this layout is the largest you will be able to fit in the width of an iPhone browser, width = 958, with a css margin of 2px.

The css pixel width of an iPhone 5s is 980, whereas the physical pixel width is 640.

The outer group of a simulations (div) should have its css position changed to relative.

Buttons and other controls should be made larger than the default in LiveCode in order to manipulate more easily in web browsers.

Next we want to experiment with smaller simulations.

We want to see if we can put text to one side or the other, or even wrap text continuously around the simulation frame.
